1. Clean
mvn clean - команда в терминале
удаляет результат предыдущей сборки
типично делают clean и потом делают --> package

------------------------------------------------------------------------------------

2. Validate
быстрая проверка, что проект валидный, есть валидный pom.xml и базовая структура

------------------------------------------------------------------------------------

3. Compile
компиляция src/main/java --> target/classes, быстрая проверка компиляции кода

------------------------------------------------------------------------------------

4. Test
запускает unit-тесты типично перед package
mvn clean package -DskipTests

------------------------------------------------------------------------------------

5. Package
собирает итоговый артефакт проекта и кладёт его в target

------------------------------------------------------------------------------------

6. Verify
проверка качества сборки, валидация артефактов (версий т.д.), плагины контроля

------------------------------------------------------------------------------------

7. Install
кладёт наш артефакт проекта в локальный репозиторий

------------------------------------------------------------------------------------

8. Site
документация и отчёты проекта (используется очень редко)

------------------------------------------------------------------------------------

9. Deploy
отправка/публикация приложения на отдалённый репозиторий (используется очень редко), делает релизную версию.
Альтернатива использование: Jenkins или Tekton

Команды
mvn dependency:tree  вывод на экран дерева зависимостей
